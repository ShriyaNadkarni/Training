precision numbers

JavaScript numbers are always stored as double precision floating point numbers . This format stores numbers in 64 bits, where the number (the fraction) is stored in bits 0 to 51, the exponent in bits 52 to 62, and the sign in bit 63 . For Integer Value the precision is about 15 digits , whereas for float values the precision is not always accurate

  1. BigInt : This can be used only for integer values , this won't work for floating values (eg : 2345678901234n here n determines bigint)
  2. Round Numbers: If you have numbers with too many decimal places, round them to a reasonable level of precision.



how are Number object different from their respective global model?

In practice, you would typically use the global functions when working with numeric values, and you may use Number object methods when you have a Number object created explicitly. However, JavaScript often performs automatic type coercion, so you might not always need to create Number objects explicitly.


The difference is that these methods are used when you're working with numbers in a special way, using the Number object. When you work with regular numbers (not wrapped in Number objects), you use the global functions like toFixed() directly on the numbers.
